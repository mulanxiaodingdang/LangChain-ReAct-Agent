
# 项目基础信息 (Project Context)
- 项目名称：LangChain ReAct Agent · 科研论文问答系统（基于 LangChain + ReAct 范式 + RAG 检索增强的科研论文问答系统 ，以 data/ 下的科研论文为知识库）
- 技术栈：Python 3.10+ / LangChain 0.3 / LangGraph 0.2 / Streamlit 1.40 / Chroma 向量库 / 硅基流动 API（DeepSeek-V3.2 + BGE-M3）/ YAML 配置驱动
- 范围限制：请只关注和处理与本项目相关的 Python/YAML/TXT 等文件，不要盲目查找或读取其他无关语言的文件。

# 核心行为准则 (Strict Rules)
1. 【禁止全局搜索】：为了节省计算资源，除非我明确下达"全局搜索"或"grep"等指令，否则绝对禁止扫描整个工作区。你只能精准读取我提供给你的特定文件路径。
2. 【先审批，后动手】：每次遇到修改代码、修复 Bug 或开发新功能的需求时，请先用简短的话告诉我你的"修改思路"以及"准备修改哪些文件的第几行"。必须等我明确回复"同意"后，你才能调用编辑工具真实修改本地代码。当前为科研论文问答系统项目，不允许为了节省token而截断单个chunk和单个论文元数据的内容，仅允许在单个chunk/在线搜索论文数据拼接为整体上下文后再进行截断。
3. 【极简输出】：不要说废话，不要道歉，不要解释基础常识。请直接给出结论或核心代码，最大程度节省冗余的输出消耗。
4. 【禁止未深入分析的回答】：在没有充分分析和理解问题的情况下，禁止直接给出解决方案或代码。必须先进行详细分析，明确问题的本质和可能的解决路径。如果基于当前知识和上下文无法给出明确结论，请直接说明无法判断，并提出可能的分析方法或需要进一步确认的信息。禁止强行根据当前知识和上下文给出不确定的解决方案，如果没办法判断具体问题，请直接说明无法判断，并提出可能的分析方法或需要进一步确认的信息。
